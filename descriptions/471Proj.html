SQL Database

In the Fall of 2017 I took an SQL database class. We started to learn SQL by first studying what relations are and how they can be different inside of a database. I was surprised by how many options you have to structure a database and the end result being the same. I was even more interested when we studied relational algebra. Throughout the semester we were given tasks to create relational diagrams for a company that had multiple tables to relate to eachother such as employee, manager, project, dependent, and department. My classmates and I went on to figure out the relational algebra of this database and improve the efficiency of the structure. Taking one step at a time, we then created the actual virtual database using the universities MSSQL server. Finally, we needed to create a website.

Creating a website with database features was new to me. I figured that PHP and HTML would be the optimal languages to use in this scenario. Even though I had no idea how to code in php, once I read about it and practiced using it I was in love with the language. PHP is so simple, yet very important and efficient. I wanted my code to be secure, so I coded my PHP scripts to be safe against any SQL injection attempts. This may have been overkill for this project, but I think it is important to always practice safe programming methods if I have to make a program for a client when I have a job.
